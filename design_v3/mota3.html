ĐẶC TẢ KỸ THUẬT · Tài liệu bàn giao phát triển

Ứng dụng “Tạo Mục lục hồ sơ lưu trữ”
Xây dựng trên .NET 8 — Windows Forms (WinForms)

Mục tiêu: tái tạo 100% giao diện và luồng nghiệp vụ của bản mẫu HTML hiện có, chạy dưới dạng ứng dụng desktop Windows. Tài liệu này mô tả kiến trúc, từng màn hình, từng control, hệ màu/typography, mô hình dữ liệu, bộ máy sinh mẫu (template engine), phân trang, xuất PDF hàng loạt và xử lý đa luồng.

Phiên bản mẫu tham chiếu: bản “HUCE” (accent xanh #0043a5). Bản gốc dùng xanh Windows #0078d7 — chỉ khác token màu, mọi thứ khác giữ nguyên.

0. Tóm tắt chức năng

Ứng dụng đọc dữ liệu văn bản/tài liệu từ file Excel, cho người dùng ghép (mapping) các cột Excel vào biến của một mẫu (template) mục lục, cho phép thiết kế/soạn thảo mẫu trực quan (WYSIWYG) trên khổ A4 có phân trang, rồi sinh hàng loạt mỗi hồ sơ thành một file PDF mục lục — chạy đơn luồng hoặc đa luồng.

Toàn bộ thao tác gói trong một wizard 3 bước: Bước 1 · Đầu vào Bước 2 · Thiết kế & Preview Bước 3 · Chạy.

2. Cửa sổ chính & khung wizard

2.1 Form chính (MainForm)

  • Tiêu đề cửa sổ (title bar): Tạo mục lục hồ sơ — SaoMai kèm icon xanh chữ “T”.
  • Kích thước khởi động gợi ý: 1200 × 780, MinimumSize 1040×680, cho phép maximize.
  • Bố cục dọc: (a) thanh bước (step header) trên cùng — cao ~64px; (b) vùng nội dung bước (fill); (c) thanh điều hướng dưới cùng — cao ~52px với nút ◀ Quay lại và Tiếp theo ▶ / Tạo Mục lục.

2.2 Thanh bước (Step header)

Ba mục nằm ngang, ngăn bởi mũi tên ›. Mỗi mục gồm badge tròn số + hai dòng chữ (nhãn nhỏ “BƯỚC n” + tên bước):

  • Dòng trạng thái ngắn bên cạnh (statusMap) mô tả bước hiện tại.

4. Mô hình dữ liệu

4.1 Bản ghi hồ sơ (record)

Mỗi hồ sơ là một đối tượng có các trường “cấp hồ sơ” + một danh sách rows (các văn bản trong hồ sơ). Trường cấp hồ sơ và cấp hàng ánh xạ 1–1 với biến mẫu:

public sealed class HoSo {
    public string SoHoSo { get; set; }          // {so_ho_so}
    public string DonVi { get; set; }            // {don_vi}
    public string ChiNhanh { get; set; }         // {chi_nhanh}
    public string TieuDe { get; set; }           // {tieu_de}
    public string NguoiLap { get; set; }         // {nguoi_lap}
    public List<VanBan> Rows { get; set; } = new();
}
public sealed class VanBan {
    public string Stt { get; set; }              // {stt}
    public string SoKyHieu { get; set; }         // {so_ky_hieu}
    public string NgayThang { get; set; }        // {ngay_thang}
    public string TacGia { get; set; }           // {tac_gia}
    public string TrichYeu { get; set; }         // {trich_yeu}
    public string ToSo { get; set; }             // {to_so}
    public string GhiChu { get; set; }           // {ghi_chu}
    public string LoaiVb { get; set; }           // {loai_vb}   (Mẫu 03)
    public string SlTrang { get; set; }          // {sl_trang}  (Mẫu 03)
}

4.2 Biến tự động

  • {trang_so} — số trang hiện tại (tính khi phân trang).
  • {tong_so_trang} — tổng số trang của hồ sơ.

Hai biến này KHÔNG lấy từ Excel; đánh dấu auto = true, hiển thị nhóm chip riêng (nền xanh lá #eefaf0, viền #7fc08a).

4.3 Header Excel mặc định (19 cột mẫu)

STT · Số, ký hiệu hồ sơ · Số, ký hiệu văn bản · Ngày tháng văn bản · Tác giả · Trích yếu nội dung VB · Tờ số · Ghi chú · Người lập · Đơn vị chủ quản · Chi nhánh · Tiêu đề hồ sơ · Thời hạn bảo quản · Số lượng tờ · Thời gian tài liệu · Tổng số văn bản · Tổng số tờ · Tình trạng vật lý · Ngày lập · (Mẫu 03 thêm: Loại văn bản, Số lượng trang).

6. Bước 1 — Đầu vào

Bố cục dạng lưới, canh cột thẳng nhau (mép phải vùng Excel thẳng mép phải cột “Cột trong Excel”, mép trái vùng Mẫu thẳng mép trái cột “Biến trong mẫu”).

6.1 Hàng đầu — hai fieldset

  • Nguồn dữ liệu Excel (bên trái): nhãn trái Tệp: + textbox readonly + nút Duyệt… (OpenFileDialog, lọc *.xlsx); nhãn Sheet: + combobox (nạp sau khi chọn file); nút Đọc dữ liệu + nhãn trạng thái (“Chưa đọc dữ liệu…” → “✓ Đã đọc N dòng · M cột”).
  • Mẫu (template) (bên phải): combobox chọn 1 trong 4 mẫu; hiển thị số biến đầu vào của mẫu.

6.2 Banner trạng thái ghép biến

Một dải thông báo đổi màu theo trạng thái: chưa đọc (vàng nhạt) → “✓ Đã ghép đủ N biến” (xanh) → “⚠ K biến chưa gán — sửa xong mới sang được bước sau” (đỏ).

6.3 Bảng ghép biến ↔ cột Excel

DataGridView (hoặc lưới tùy biến) 5 cột: # · Cột trong Excel (header) · → · Biến trong mẫu · Tự khớp.

  • Bên phải (Biến trong mẫu) cố định theo mẫu đang chọn — hiển thị dạng chip {ten_bien}.
  • Bên trái (Cột Excel) là combobox cho mỗi biến; trước khi “Đọc dữ liệu” thì mờ/disable ghi “— chưa đọc dữ liệu —”. Sau khi đọc: tự khớp theo tên gần đúng, đổi được tay. Danh sách gồm — (chưa gán) — + tất cả header.
  • KHÔNG kiểm tra trùng cột (cho phép nhiều biến dùng chung 1 cột, không cảnh báo).
  • Cột “Tự khớp”: “✓ khớp” (xanh) hoặc “⚠ chưa gán” (vàng).
  • Có thanh cuộn dọc luôn hiện (reserve gutter) để canh cột ổn định.

6.4 Cột gom nhóm hồ sơ

Fieldset nhấn (viền trái accent). Combobox chọn cột dùng để tách file (mặc định tự chọn cột “Số, ký hiệu hồ sơ” sau khi đọc). Bên cạnh hiển thị số file PDF sẽ tạo theo lựa chọn hiện tại — ví dụ “→ sẽ tạo 1.000 file PDF (mỗi giá trị của cột này 1 file)”, hoặc chọn “(không gom nhóm)” → “→ gộp tất cả hồ sơ vào 1 file PDF”.

6.5 Điều kiện hợp lệ để sang Bước 2

step1Valid = dataRead == true && (số biến chưa gán == 0). Khi chưa hợp lệ: nút “Tiếp theo ▶” disable.

7. Bước 2 — Thiết kế & Preview

Hai vùng: trái = tờ A4 (soạn thảo/xem trước) trên nền xám; phải = panel danh sách biến. Trên cùng có tab Chỉnh sửa / Xem trước, thanh điều hướng hồ sơ (◀ / “Hồ sơ {so} (i/n)” / ▶) và (ở bản này) nút xuất nằm hợp lý.

7.1 Thanh công cụ định dạng (chỉ hiện ở chế độ Chỉnh sửa)

Một hàng, thứ tự từ trái sang phải:

  1. Font (combobox: Times New Roman, Arial, Calibri, Tahoma, Segoe UI, Cambria, Verdana) — mặc định Times New Roman.
  2. Cỡ chữ (combobox: 9,10,11,12,13,14,16,18,20,24) — mặc định 13.
  3. | B đậm · I nghiêng · U gạch chân.
  4. | A− giảm cỡ · A+ tăng cỡ.
  5. | Căn trái · giữa · phải (icon).
  6. | − zoom out · % · + zoom in (50%–200%, áp cho tờ A4 ở cả 2 chế độ) · ⇄ A4: Dọc/Ngang (đổi hướng trang).
  7. | ➕ Chèn cột · ✖ Xóa cột — chỉ hiện khi con trỏ đang focus vào một ô trong bảng; đưa con trỏ ra ngoài bảng thì tự ẩn. · ↶ Hoàn tác (undo thao tác cột).

Kích thước nút nhỏ (cao 22–26px, chữ 11–12px), khoảng cách nút hẹp (gap ~3px). Bỏ nhãn “Font:” chữ ở đầu.

7.2 Chèn / xóa / co giãn cột trong bảng

  • Bấm vào ô bất kỳ → chọn cột đó (highlight viền xanh), hiện nút Chèn/Xóa.
  • Chèn cột: thêm cột mới bên phải cột đang chọn (header trống + ô thân trống), chuẩn hóa lại bề rộng.
  • Xóa cột: xóa cột đang chọn.
  • Kéo mép cột (col-resize khi chuột gần mép phải header, ngưỡng 7px) để co giãn; cột kế bù trừ; lưu bằng % để giữ tỉ lệ.
  • Mọi thao tác cột đẩy vào stack Undo (giới hạn ~40).

7.3 Panel biến (bên phải)

  • Biến gom theo nhóm: Tài liệu (hồ sơ) và Hàng (bảng văn bản), cộng nhóm tự động ({trang_so},{tong_so_trang}).
  • Chế độ Chỉnh sửa: bấm chip → chèn biến tại vị trí con trỏ (một <span data-var> không cho sửa nội bộ; xóa thì xóa cả biến, không xóa từng ký tự).
  • Chế độ Xem trước: chip KHÔNG bị mờ; bấm chip → nổi sáng vàng giá trị biến đó trong trang (biến hàng → sáng cả cột kể cả tiêu đề; biến đơn → sáng đúng ô). Bấm giá trị trong bảng cũng nổi sáng cả cột. Bấm lại để tắt.
  • Nút Ghép biến với dữ liệu (mở popup bảng mapping giống Bước 1).
  • Khi mẫu đã chỉnh sửa (dirty): hiện nút Lưu và Reset về mặc định.

8. Template engine & phân trang

Đây là phần lõi. Mỗi mẫu là một khối HTML gồm: phần đầu (header) → một bảng (thead + một hàng mẫu data-loop) → phần chân (footer) → node số trang. Thuật toán resolve + phân trang:

  1. Thay mọi <span data-var="k"> bằng giá trị thật valFor(record, k, pageNo, total). Với biến suy diễn: loai_vb đoán từ tiền tố trích yếu (Quyết định/Tờ trình/Thông báo/Công văn/Giấy chứng nhận/Biên bản/Đơn/Phiếu/Hợp đồng/Trích lục → mặc định “Văn bản”); sl_trang = max(1,(toSo%4)+1) (chỉ là quy tắc mẫu — trong app thật lấy từ cột Excel nếu có).
  2. Nhân bản hàng data-loop cho từng văn bản trong rows.
  3. Đo chiều cao thật ở bề rộng nội dung theo hướng trang (portrait/landscape) để chia các hàng vào từng trang sao cho không tràn (đo header, từng hàng, footer, node số trang).
  4. Sinh nhiều “tờ” (page card): tờ đầu có header; các tờ có bảng (thead lặp lại mỗi trang) + phần hàng thuộc trang; tờ cuối có footer (nếu đủ chỗ, nếu không thì footer sang trang riêng); mỗi trang có node số trang cập nhật {trang_so}/{tong_so_trang}.
  5. Khung nội dung mỗi tờ là flex-column, min-height = chiều cao vùng nội dung, để node số trang (margin-top:auto) rơi sát đáy góc phải.
Cách hiện thực trong .NET để “giống 100%”: Giữ nguyên chuỗi HTML template + thuật toán trên, chạy trong WebView2 (đo chiều cao bằng JS getBoundingClientRect). Khi xuất PDF, nạp cùng HTML đã resolve vào PuppeteerSharp và page.PdfAsync khổ A4. Như vậy layout preview và PDF trùng khớp tuyệt đối. (QuestPDF là phương án thuần .NET nhưng phải lập trình lại toàn bộ bố cục & luật phân trang.)

9. Bước 3 — Chạy & xuất PDF

9.1 Khung “Thư mục & tên file xuất”

  • Thư mục: textbox readonly + nút Duyệt… (FolderBrowserDialog). Thư mục xuất chỉ ở Bước 3.
  • Tên PDF: combobox mẫu sẵn + textbox tự do. Hỗ trợ mọi biến của mẫu, cộng biến đặc biệt: {stt_file} (số thứ tự tăng dần, 3 chữ số), {ngay_gio} (ngày giờ YYYYMMDD_HHmm). Preset gợi ý: MLHS_{so_ho_so}.pdf, {stt_file}_{so_ho_so}.pdf, MLHS_{ngay_gio}_{so_ho_so}.pdf, {so_ho_so}_{nguoi_lap}.pdf, PDF_{stt_file}.pdf. Hiển thị ví dụ tên tính trực tiếp; tự thay ký tự cấm \ / : * ? " < > | bằng _; tự thêm .pdf.

9.2 Khung “Tùy chọn chạy”

  • ☑ Chạy đa luồng (song song — nhanh nhất). Bỏ chọn = đơn luồng. Ẩn ô nhập số luồng (tự quản lý).
  • ☑ Ghi đè file đã tồn tại.
  • ☑ Xuất kèm file Excel tổng hợp.
  • ☑ Bỏ qua hồ sơ lỗi (không dừng cả mẻ).
  • Dòng ⏱ Ước tính thời gian theo chế độ (đa luồng ~48 giây / đơn luồng ~3 phút 30 cho ~1.000 hồ sơ — trong app thật tính động).

9.3 Nút chạy & tiến trình

  • Chỉ một nút hành động: Tạo Mục lục (nằm ở thanh dưới; đang chạy “Đang tạo…”, xong “Tạo lại”). Không còn nút “Tạo PDF hàng loạt”/“Bắt đầu sinh HTML” riêng.
  • Trên thanh tiến trình: bên trái chữ thường “Tiến trình chạy (x/y)” — x = số hồ sơ đã tạo, y = tổng; chỉ hiện khi đang chạy hoặc đã chạy xong (xong vẫn hiện y/y); bên phải “Hoàn tất”.
  • Thanh progress bar (nền #e6e6e6, fill accent).
  • Console log nền tối, cuộn theo: mỗi hồ sơ “✓ Hồ sơ {so} → {tên file}.pdf”; lỗi “✗ … LỖI: … → bỏ qua/DỪNG”; kết thúc “✔ Hoàn tất: N PDF • t giây • K lỗi”.
  • Sau khi xong: nút 📁 Mở thư mục kết quả; nếu có lỗi: “⚠ K hồ sơ lỗi” + nút ↻ Thử lại hồ sơ lỗi.

  • “Thử lại hồ sơ lỗi”: chạy lại danh sách lỗi đã gom.
